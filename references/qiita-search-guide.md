# Qiita 検索ガイド

## なぜ Qiita を美容アイデアリサーチに使うのか

Qiita は日本語技術記事のプラットフォームで、海外ソース（Reddit・HN）では拾えない
**日本市場特有の視点**を補完します。

- 日本の開発者が実際に作っている美容テック系プロダクト・ツールの動向
- 日本語ユーザー向け美容アプリの UX 課題・技術スタックの議論
- 日本語圏でのスキンケア・コスメ・ヘルスケア領域の API・データセット情報
- 海外スタートアップにはない日本の商慣行・規制を踏まえた事業考察

Reddit = **ユーザーの生の痛み**、HN = **ビルダー・投資家の視点（英語圏）**、
Qiita = **日本語ビルダー・エンジニアの視点** として三角補完します。

## 日付範囲ロジック

`fetch_qiita.py` は以下のルールで検索期間を自動決定します：

| 条件 | 検索期間 |
|------|---------|
| `--year` が現在年（例：2026年に `--year 2026`） | 今日から過去365日（過去1年分） |
| `--year` が過去年（例：2025年以前） | 指定年の1月1日〜12月31日 |

例：2026年3月に `--year 2026` を指定 → 2025年3月〜2026年3月を検索

## Qiita API 認証

`.env` ファイルに `QIITA_TOKEN` を記載することで、レートリミットが緩和されます。

```
QIITA_TOKEN=your_token_here
```

- **トークンあり**: 1000リクエスト/時
- **トークンなし**: 60リクエスト/時

トークンは [Qiita アカウント設定 > アプリケーション](https://qiita.com/settings/applications) から発行できます。

---

## エンゲージメントスコアの計算式

Qiita では「ストック（保存）」が品質の高い記事の指標として機能します。

```
engagement_score = いいね数 + ストック数 × 2 + コメント数 × 3
```

## 手動検索のヒント

スクリプトで結果が少ない場合、Qiita サイト上で直接検索できます。

1. [qiita.com/search](https://qiita.com/search?q=スキンケア+アプリ) を活用
2. タグ検索: `tag:スキンケア`, `tag:美容`, `tag:コスメ`
3. 投稿日フィルタ: `created:>=2025-03-01 created:<=2026-03-01 スキンケア`

## Reddit・HN との組み合わせ方

強い候補アイデアは **3つのソース全てにシグナルがある** ものです。

| ソース | シグナルの種類 |
|--------|-------------|
| Reddit | ユーザーの生の不満・要望 |
| HN | 英語圏のビルダー・投資家の動向 |
| Qiita | 日本語圏エンジニアの実装・関心 |

例：
- Reddit: 「成分をスキャンして安全か教えてくれるアプリが欲しい」
- HN: 「Ask HN: ingredient safety checker for cosmetics」
- Qiita: 「コスメの成分解析 API を作ってみた」

→ 三方向のシグナルが揃えば、日本市場への特化で差別化できる強いアイデア。
